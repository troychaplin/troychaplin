## The concept

A WordPress block theme for a frontend developer's blog and portfolio, built around the metaphor of a manual telephone switchboard. Posts are connections being made. Tags are lines. The chrome plays the role of operator and equipment.

For eventual WordPress directory submission, so the metaphor should be strong enough to be memorable but loose enough that someone else's content could live in it.

## Where the weirdness lives

**Navigation as patching.** Plug-and-jack imagery, line indicators, connection metadata, operator-panel UI. That's the loud channel.

Typography, color, and motion stay restrained so the navigation reads as intentional, not gimmicky.

## Anchors, not a checklist

- "Connection" or "call" as a name for posts
- Tag treatment as "lines"
- A homepage that reads as a board
- Plug-and-jack visual language as link or divider treatments
- An archive that reads like a call log
- A palette of bakelite, brass, signal-light amber, and operator-panel green

## What I want from you first

Before any code:

1. **Pick an era.** 1920s manual cord boards? 1950s electromechanical exchanges? 1970s PBX? Each has a totally different look.
2. **Typographic direction**, with a reason — operator panels have specific typographic histories.
3. **A named palette** — colors named in the language of the exchange.
4. **Three or four block patterns** that couldn't exist in any other theme.
5. **A first sketch of the homepage.**

Show me thinking before code. I want to react to direction before we commit.

## Constraints

- WordPress block theme, theme.json driven
- Directory-submission ready (GPL-compatible fonts, accessibility, no proprietary anything)
- Light and dark modes both — switchboards run all night
- The metaphor never breaks character, but it never winks at the camera either

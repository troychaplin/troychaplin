## The concept

A WordPress block theme for a frontend developer's blog and portfolio, built around the metaphor of a recipe card box. Posts are cards. The archive is the box. Tutorials become literal recipes; experiments become "notes from this attempt." Warm, structured, useful.

For eventual WordPress directory submission, so the metaphor should be strong enough to be memorable but loose enough that someone else's content could live in it.

## Where the weirdness lives

**Card-based layout and structured metadata.** Recipe-card proportions, ingredient/method/notes structure, file-card edges. That's the loud channel.

Type and color stay disciplined so the card structure reads as content, not decoration.

## Anchors, not a checklist

- Posts laid out with recipe-card proportions in some views
- "Ingredients," "method," "notes from this attempt" as structural blocks
- Handwriting-accent typography used sparingly (one hand, used well)
- A "card box" archive view
- A palette of stained paper, ink, recipe-card cream, and one stain-color accent

## What I want from you first

Before any code:

1. **Pick a kitchen.** 1950s suburban Americana? Julia Child's mid-century? Contemporary professional kitchen? Family-heirloom recipe box passed down? Each is a different voice.
2. **Typographic direction**, with a reason — handwritten accents are easy to overdo; one hand done well beats three done casually.
3. **A named palette** — colors named in the language of the kitchen.
4. **Three or four block patterns** that couldn't exist in any other theme.
5. **A first sketch of the homepage.**

Show me thinking before code. I want to react to direction before we commit.

## Constraints

- WordPress block theme, theme.json driven
- Directory-submission ready (GPL-compatible fonts, accessibility, no proprietary anything)
- Light and dark modes both — kitchens run dawn to late night
- The metaphor never breaks character, but it never winks at the camera either
